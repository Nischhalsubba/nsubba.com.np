MAS DataHub

Enterprise platform UX for data integration and Maximo automation workflows. Streamlined complex data operations into a unified dashboard experience.

Outcome

Unified automation workflow

THE PROBLEM

Disjointed data workflows creating operational bottlenecks.

Enterprise users were struggling to manage data integration and Maximo automation workflows. The existing process involved fragmented tools, leading to high cognitive load, manual errors, and a steep learning curve for new operators.

"We need a way to visualize complex data flows without overwhelming the user." — Stakeholder feedback

Goals & Success Metrics

Goal
Reduce Task Time

Streamline navigation to access key data reports faster.

Goal
System Consistency

Establish a reusable design system for future modules.

Goal
Reduce Errors

Clearer feedback loops for data automation failures.

Constraints & Reality Check

  • Complex Legacy Data: Must support existing Maximo data structures without breaking changes.
  • Technical Feasibility: Design had to be implemented within a tight 3-month window by a small engineering team.
  • Data Density: Users need to see large datasets at a glance; whitespace had to be balanced with density.
INFORMATION ARCHITECTURE

Simplifying the Automation Flow

I mapped out the user journey to identify redundant steps. The new flow consolidated setup and monitoring into a single dashboard view, reducing clicks to action by 40%.

[Flow Diagram: Login -> Dashboard Overview -> Data Source Selection -> Automation Config -> Success]
DESIGN SYSTEM

Built for Scalability

Component Library

I established a modular component library focusing on data tables, status indicators, and navigation panels. This ensured consistency across the platform.

  • Standardized Data Tables with filtering/sorting
  • Status Pills (Success, Pending, Error)
  • Collapsible Sidebar Navigation
Input Field: [ Enter Data source... ]

Key Design Decisions

Decision: Side Navigation vs. Top Navigation

Why: Enterprise screens are data-heavy and wide. Vertical space is precious.

Outcome: Chose a collapsible side navigation to maximize horizontal real estate for data tables while keeping navigation accessible.

Decision: Inline Editing for Tables

Why: Users frequently need to make small updates to data records.

Outcome: Implemented inline editing to prevent context switching (modals), speeding up data entry workflows.

Final Implementation

The final delivery included high-fidelity screens, interactive prototypes for complex flows, and a redlined spec document for engineers.

Results & Impact

Consistent

Scalable
